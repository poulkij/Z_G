## 技术细节

### 路由逻辑架构

```
用户问题
    ↓
Step 1: 问题分类与路由
    ↓
┌─────────────────────────────────────────────────────────┐
│  股票/投资问题 → Step 1.5（个股问诊） → Step 2 → Step 3  │
│  人生/职业问题 → 四个圈检查 → life-decision.md           │
│  创业/商业问题 → 时代主线检查 → business-judgment.md     │
└─────────────────────────────────────────────────────────┘
    ↓
Z 哥式回答
```

### 模糊匹配规则

- **上下文优先**：如果用户之前讨论的是股票问题，后续问题默认路由到股票/投资
- **关键词权重**：核心关键词（如"股票"、"人生"、"创业"）权重 > 辅助关键词
- **意图识别**：如果用户表达困惑/迷茫，优先路由到人生/职业决策
- **领域切换检测**：如果用户明确提到不同领域的话题，切换到对应路由
